# Commit Convention

## 형식

```
<type>: <subject>

[body]

[footer]
```

## Type

| 타입 | 설명 |
|------|------|
| `feat` | 새로운 기능 추가 |
| `fix` | 버그 수정 |
| `docs` | 문서 수정 |
| `style` | 코드 포맷팅, 세미콜론 누락 등 (기능 변경 없음) |
| `refactor` | 코드 리팩토링 (기능 변경 없음) |
| `test` | 테스트 코드 추가 또는 수정 |
| `chore` | 빌드 설정, 패키지 업데이트 등 기타 변경 |

## 규칙

- subject는 50자 이내로 작성
- subject 끝에 마침표(.) 사용 금지
- subject는 명령문으로 작성 (예: "변경한다" → "변경")
- body는 선택사항이며, 변경 이유와 내용을 작성
- footer는 선택사항이며, 관련 이슈 번호를 작성

## 예시

```
feat: 로그인 기능 추가

사용자 이메일/비밀번호 기반 로그인 구현

Closes #12
```

```
fix: 회원가입 시 이메일 중복 검사 오류 수정
```

```
docs: README 설치 방법 업데이트
```
